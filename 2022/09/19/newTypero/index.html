3.13 滚动区域的bug

  • better-scroll的区域是根据scrollHeight决定的
    • 是根据better-scroll的content子组件的高度决定,最初没有将图片计算在内，后来图片加载后，有了新的高度，scrollerHeight未更新，造成的。
  • 如何解决
    • 监听每张图片是否加载完成，只要有一张加载完成，就执行一次refersh()
    • 监听图片加载完成的方法
      • 原生js通过img.onload = function(){}
      • vue通过@load='方法'等待图片加载完成，然后通过发送请求
        • 发送请求的三种方法
          1. 通过子传父：GoodsListItem->GoodsItem->Home
          2. 通过vuex - store来定义
          3. 通过原型数组:Vue.prototype.$bus=new Vue() 又称事件总线
            1. GoodsListItem通过this.$bus.$emit(事件名称， 参数)来发送请求
            2. Home通过this.$bus.$on('函数名称', () => {})来接收

3.14 防抖debounce / 节流throttle

  1. debouce:是就是指触发事件后，在 n 秒内函数只能执行一次，如果触发事件后在 n 秒内又触发了事件，则会重新计算函数延执行时间。
  • debouce(func, delay) {
     let timer = null
 
     return function(...args) {
       if(timer) clearTimeout(timer)
       timer = setTimeout(() => {
         func.apply(this, args)//func = this.$refs.scroll.refresh.apply(this, args)
       }, delay);
     }
   },

  1. throttle:节流就是每隔n的时间调用一次函数，而不是一触发事件就调用一次，这样就会减少资源浪费。

    • ```js
      function throttle(func, wait) {
      let timeout;

      return function() {
      const context = this;
      let args = arguments;
      if (!timeout) {
      timeout = setTimeout(function(){
      timeout = null;
      func.apply(context, args)
      }, wait)
      }

      }
      }

4.9首页详情页监听全局图片加载时间[mixins的使用]

  • 产生的原因：由于两个页面都使用了公共组件，来等待图片加载完接收加载后的图片，进行刷新。所以导致了，在详情页中，调用了首页的图片加载刷新。
  • 解决办法：
    • 一：在goodsListItem页面中，通过this.$route.path.index('/地址') !== -1 来判断调用不同的页面刷新
    • 二：给每个页面都添加一个注销事件总线:this.$bus.off('name', function),由于两者都有相同的代码块，所以进行封装:common文件夹下新建mixin.js,封装挂载代码，并将它从主页面导入；并且新增mixins属性，具体格式为:mixins:[]

4.10点击标题滚动到对应内容

  • 子传夫，获取导航的索引
  • 在主页面定义themeTopY数组，用于存放四个定位的高度
  • 给每个标签设置ref属性，通过this.$refs.name.$el.offsetTop获取数组的高度
  • 使用this.$nextTick，回调参数返回。弊端：也会返回多次
  • 最终解决方法：
    • data定义数组及函数，分别用来存放读取的数值及调用防抖函数
    • 在页面创建完毕的时候，会使用防抖函数，来定义函数，保证它只调用一次。然后在图片加载完毕的时候，使用这个函数。至此完成
